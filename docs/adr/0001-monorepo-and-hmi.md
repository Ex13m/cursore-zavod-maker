# ADR-0001: Монорепо, Factory HMI, автопилот на GitHub Actions + Netlify

Дата: 2026-07-08 · Статус: принято (опрос владельца)

## Контекст

Нужен автономный завод курсоров: производство по расписанию, тест владельцем,
публикация на Gumroad. Инфраструктура — бесплатная, репозиторий публичный.

## Решение

1. **Монорепо** (`packages/cursorfx` + `factory` + `portal`): один cron, один деплой,
   атомарные изменения через все слои. Альтернатива (портал отдельно, библиотека как
   зависимость) отклонена: двойная синхронизация ломает автономность.
2. **Дизайн Factory HMI**: цвет только у статусов, графитовая гамма, никакой
   декоративной анимации — движется только курсор. Тематически совпадает с «заводом»,
   выделяется на фоне glass/brutalism-мейнстрима 2026.
3. **Автопилот: GitHub Actions (cron 07:00 UTC) + Netlify**: работает без включённого
   ПК, $0. Альтернативы: локальный планировщик (не автономно), облачный Claude-агент
   (расход квот; оставлен как опция для тренд-анализа).
4. **Git = база данных**: дропы и журнал — коммиты в `factory/data/`. Бесплатный аудит,
   откат, история; масштаба одного владельца хватает с запасом.

## Последствия

- Плюс: нулевая инфраструктура, полная воспроизводимость (сид = дата).
- Минус: ежедневные коммиты дропов шумят в истории — приняли (это и есть журнал).
- Минус: серверлес-очередь живёт в Netlify Blobs, а не в git — синхронизация
  публикаций с журналом делается на стороне функций.
