# ADR-0002: Web-first; сценарные курсоры как премиум; Windows .ani — экспорт позже

Дата: 2026-07-08 · Статус: принято

## Контекст

Вопрос владельца: продавать курсоры только для сайтов или сразу под Windows/Mac?

## Факты платформ

- Системный курсор Windows (`.cur`/`.ani`) — маленький покадровый спрайт. Сцена вокруг
  указателя (ракета-преследователь, турели, стая рыб, звук) физически невозможна.
- macOS не разрешает менять системный курсор (только хак Mousecape) — рынка нет.
- В браузере ограничений нет: canvas-сцены, физика, WebAudio.

## Решение

1. **Web-first**: основной товар — код-пресеты для сайтов. Сценарные курсоры
   (`turret`, `rocket`, `lure`, `noiseblob`) — премиум-сегмент, недоступный
   системным курсорам; звук — генерируемый (WebAudio), opt-in.
2. Сценарные эффекты наследуют общий `CanvasEffect` (полноэкранный canvas, DPR),
   остаются сериализуемыми `EffectSpec` — конвейер завода их производит как данные.
3. **Windows `.ani` — M3**: экспортёр кадров для простых эффектов расширит каталог
   на рынок Etsy/Gumroad-паков. Mac — не делаем.

## Последствия

- ОТК: contrast-проверка не применяется к сценарным эффектам (сцена самосветящаяся).
- Звук по умолчанию выключен (автовоспроизведение, вежливость); тумблер в карточке.
- Превью в карточке рисуется статически (portal/src/preview.ts) для каждого типа.
