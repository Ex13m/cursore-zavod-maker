# CURSOR ZAVOD — Спецификация продукта v1.0

> Автономный завод анимированных курсоров: анализ трендов → производство → тест → продажа.
> Статус: **утверждено к разработке** · Дата: 2026-07-08 · Владелец: Ex13m

---

## 1. Видение

**Cursor Zavod** — автономный производственный портал, который сам, по расписанию,
анализирует дизайн-тренды, производит партии анимированных веб-курсоров, присылает
владельцу письмо со ссылкой на «пульт управления», где владелец тестирует курсоры
вживую, одобряет галочкой — и одобренное автоматически уходит на площадку продаж
(Gumroad). Дополнительно курсор можно создать вручную — через чат на портале,
описав его словами.

**Принцип:** exception-based review — завод работает сам, человек только
одобряет и разруливает тревоги.

## 2. Рынок и модель монетизации

- Кастомные курсоры продаются: Gumroad/Etsy — паки $0.99–$15; кодовые
  компоненты — Motion+ (подписка), отдельные эффекты на Gumroad.
- Модель: **open-core** — библиотека `cursorfx` бесплатна (npm, охват),
  продаются паки пресетов (zip: конфиг + сниппет + обложка + превью).
- Производительность завода: 10 курсоров/день = ~300/мес; при конверсии
  продаж даже 2–3 % каталог становится значимым пассивом.

## 3. Персоны

| Персона | Потребность |
|---|---|
| **Владелец (Ex13m)** | Утреннее письмо → 10 минут на тест/одобрение → пассивный доход |
| **Покупатель-разработчик** | Скачал zip → `createCursor(config.effects)` → работает |
| **Клод-агенты завода** | Чёткие правила в `.claude/`, скилы, воркфлоу |

## 4. Архитектура (монорепо)

```
cursore-zavod-maker/
├─ CLAUDE.md              # правила проекта (<200 строк)
├─ AGENTS.md              # реестр агентов завода
├─ .claude/               # rules / skills / agents / workflows / hooks
├─ docs/                  # спека, PDF, ADR, ассеты
├─ packages/cursorfx/     # библиотека эффектов (ядро, 0 зависимостей)
├─ factory/               # автономное производство (Node)
│   ├─ trends.js          # тренд-агент: сигналы → trends.json
│   ├─ produce.js         # генератор: тренды → дроп дня (batch JSON)
│   ├─ chat.js            # промпт → EffectSpec (эвристика + Anthropic)
│   ├─ imagegen.js        # Replicate FLUX → image-курсоры
│   ├─ publish.js         # очередь → Gumroad / локальный бандл
│   └─ notify.js          # письмо со ссылкой на пульт (Resend)
├─ portal/                # HMI-дашборд (Vite, статика + serverless)
└─ .github/workflows/     # ci.yml + daily-factory.yml (cron 07:00 UTC)
```

**Поток данных:** `factory/data/*.json` — единственное состояние (коммитится
в репо; git = база данных и журнал аудита). Портал читает JSON статически,
пишет через serverless-функции Netlify.

## 5. Функциональные требования

### F1. Автопилот (daily-factory)
- F1.1 Cron 07:00 UTC: `trends → produce → commit → deploy → email`.
- F1.2 Тренд-агент собирает сигналы (палитры, стили, теги) в `trends.json`
  с весами; источники расширяемы; фолбэк — встроенный банк трендов 2026.
- F1.3 Генератор производит 10 курсоров/день: архетипы × трендовые палитры,
  сид = дата (детерминизм). Batch → `factory/data/drops/YYYY-MM-DD.json`.
- F1.4 Письмо владельцу (Resend): ссылка на пульт + миниатюры имён.

### F2. Пульт (портал, Factory HMI)
- F2.1 Экран «Пульт»: статус линии, дата дропа, датчики трендов, тревоги.
- F2.2 Конвейер: карточки дропа дня; live-тест курсора на странице (TEST).
- F2.3 Одобрение: галочка → очередь публикации; kill-switch на партию.
- F2.4 Склад: все прошлые дропы, фильтры по тегам/статусам.
- F2.5 Цех-чат: текстовый промпт → курсор (F3), сразу live-тест.
- F2.6 Продажи: список опубликованного со ссылками Gumroad.

### F3. Чат-производство (промпт → курсор)
- F3.1 Эвристический парсер (без ключа): цвета, эффекты, скорость, стиль
  из текста промпта → EffectSpec.
- F3.2 С ключом Anthropic: Claude превращает промпт в полный EffectSpec
  (структурированный вывод, валидация схемой).
- F3.3 С ключом Replicate: «нарисуй курсор-дракона» → FLUX генерирует
  спрайт → ImageCursor с анимацией (spin/pulse).

### F4. Публикация
- F4.1 Очередь: queued → publishing → published/error (журнал в git).
- F4.2 Gumroad-провайдер: загрузка zip в draft-товар по product_id, enable.
  Ограничение платформы: API не создаёт товары (issue antiwork/gumroad#4019)
  — черновики создаются владельцем заранее, пул product_id хранится в конфиге.
- F4.3 Фолбэк: локальный бандл в `published/` + инструкция.
- F4.4 Бандл: `cursor.config.json` + README + cover.png (авто-рендер).

## 6. Нефункциональные требования

| Требование | Значение |
|---|---|
| Автономность | 0 действий владельца для производства; только одобрение |
| Стоимость инфраструктуры | $0 (GitHub Actions free tier + Netlify free) |
| Деградация | Каждый внешний ключ опционален; всё работает без ключей |
| Производительность портала | ≤ 100 KB JS (без AI-вызовов в клиенте), 60 fps курсоры |
| Секреты | Только GitHub Secrets / Netlify env; в коде — {ENV_VAR} |
| Аудит | Каждый дроп/публикация — отдельный commit с журналом |

## 7. Дизайн-система «Factory HMI»

- **Фон:** графит `#1a1d21`; панели `#22262b`; линии `#2e343b`.
- **Цвет только у статусов:** RUN `#2ecc71`, QUEUED `#f39c12`,
  ALARM `#e74c3c`, INFO `#3498db`. Всё остальное — оттенки серого.
- **Типографика:** индустриальный гротеск (Archivo/IBM Plex Sans) +
  моно для данных (IBM Plex Mono).
- **Метафоры:** конвейер (лента карточек), манометры (датчики трендов),
  сигнальные лампы, шильдики, kill-switch, штамп «ОДОБРЕНО».
- **Правило high-performance HMI:** внимание оператора — только на
  отклонения; никакой декоративной анимации вне курсоров.

## 8. Агенты завода (AGENTS.md)

| Агент | Роль | Запуск |
|---|---|---|
| trend-scout | сбор и взвешивание тренд-сигналов | cron / вручную |
| producer | генерация дропа по трендам | cron / вручную |
| qa-inspector | валидация схем, лимитов, контраста | в конвейере |
| publisher | упаковка + Gumroad | по одобрению |
| notifier | письма владельцу | после produce |

## 9. Дорожная карта

- **M1 (сейчас):** монорепо, библиотека, фабрика, HMI-портал, cron, письмо.
- **M2:** Replicate-спрайты в чате, Anthropic-чат, Netlify Functions для
  live-одобрения с телефона.
- **M3:** авто-A/B цен, витрина-магазин своя, аналитика продаж на пульте.

## 10. Риски

| Риск | Митигация |
|---|---|
| Gumroad API не создаёт товары | пул черновиков + локальный фолбэк |
| AI-слоп (низкое качество дропов) | qa-inspector + exception review владельцем |
| Ключи утекут в публичный репо | .env только локально, secrets в CI, gitleaks-паттерны в правилах |
| GitHub Actions cron дрейфует | idempotent produce (сид = дата), пропуск догоняется |

## 11. KPI

- ≥ 10 курсоров/день без вмешательства; письмо ≤ 07:10 UTC.
- Время владельца ≤ 10 мин/день.
- ≥ 95 % дропов проходят qa-inspector без тревог.
