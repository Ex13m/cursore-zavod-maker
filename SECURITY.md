# Безопасность CURSOR ZAVOD

## Модель угроз

Репозиторий **публичный**, портал **публичный**, владелец — один (Ex13m).
Главные риски: утечка API-ключей, злоупотребление публичными эндпоинтами
(расход платных токенов, мусор в очереди), supply chain (зависимости).

## Секреты

- В коде ключей нет и не будет: только `{ENV_VAR}`-ссылки. Проверка — hook
  `.claude/hooks/guard-secrets.sh` блокирует запись строк, похожих на ключи.
- Локально: `.env` (в `.gitignore`). CI: GitHub Secrets. Прод: Netlify env vars.
- Ключи: `ZAVOD_ADMIN_KEY`, `GUMROAD_ACCESS_TOKEN`, `ANTHROPIC_API_KEY`,
  `REPLICATE_API_TOKEN`, `RESEND_API_KEY`.

## Доступ к API портала

- Чтение (дропы, очередь, health) — публичное: данных чувствительных нет.
- **Мутации и AI-чат — только с заголовком `X-Zavod-Key`** равным
  `ZAVOD_ADMIN_KEY` (иначе 401). Без переменной — открытый режим,
  допустим ТОЛЬКО локально. На проде переменная обязательна.
- Ключ вводится кнопкой КЛЮЧ на пульте, хранится в localStorage браузера владельца.

## Зависимости

- `npm audit` в норме: 1 low (dev-цепочка). Прод-зависимости фабрики: только
  `archiver`, `@netlify/blobs` (10.1.0, зафиксирован). Библиотека cursorfx —
  0 зависимостей.
- Обновления — вручную с прогоном typecheck/build/smoke.
