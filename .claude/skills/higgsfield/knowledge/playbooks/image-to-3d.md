# Рецепт: объект/товар из картинки → 3D-меш (GLB)

**Задача:** превратить фото объекта в 3D-модель (GLB), опц. с текстурой, PBR, риггингом/анимацией.

**Выбор модели:**
- один ракурс, общий случай → `image_to_3d` (текстура/PBR/риггинг опционально);
- есть 2–4 ракурса одного объекта → `multi_image_to_3d` (точнее геометрия);
- нужно вытащить один конкретный объект со сложного снимка → `sam_3_3d` (+ `prompt`
  с названием объекта для дизамбигуации);
- уже есть 3D-модель, нужен скелет → `3d_rigging` (вход `model_url` = job_id прошлой
  3D-генерации или https GLB-URL, НЕ картинки).

**Ключевое правило:** меш воспроизводит ТОЛЬКО то, что есть на исходной картинке.
Добавить/сменить пропсы, одежду, предмет в руке → сперва отредактировать картинку
`generate_image`, потом конвертировать результат.

**Шаги (MCP):**
1. Вход: `media_upload_widget` (локально) или `media_import_url` (веб-URL) → media_id.
2. `models_explore(type:'3d', action:'get', model_id:…)` — посмотреть `medias[].roles` и параметры.
3. `generate_3d(model:…, params:{medias:[…], …})`; `get_cost:true` для preflight.
4. Анимация: `animation_actions` (найти клип) → повтор с `animation_action_id` +
   `enable_animation:true` (для риггованной модели).
5. Опрос `job_status sync:true` → скачать GLB из `result_url`.

**Проверка:** геометрия без дыр, текстура на месте; для риггинга — скелет корректен.
