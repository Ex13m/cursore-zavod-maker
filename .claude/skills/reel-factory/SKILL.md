# REEL FACTORY — видео-конвейер (любые форматы)

Собирает готовый ролик из AI-клипов. Выжимка боевого пайплайна UNDERGROUND FACTORY:
десятки собранных роликов, все грабли учтены ниже.

## Форматы (правила одни, меняются рамка и темп)

- **9:16, 720×1280** — рилсы/шортсы/клипы: нарезка ~2 с, титры крупно, CTA-плашка.
- **16:9, 1280×720** — hero-фоны сайта, ютуб, промо: клипы длиннее (3–5 с),
  титры мельче или совсем без них (фоновое видео = без текста и БЕЗ звука, `-an`);
  для фоновых — пережатие CRF 25 без аудио.
- **1:1, 1024×1024** — посты-квадраты: как 9:16, но titles y=h*0.78.
- **Интро-заставка** — один эффектный клип + консольный/HUD-слой поверх делает сам сайт;
  ключевой момент (вспышка/удар) — на фиксированной секунде, сообщить её разработчику.
- **Сборник-компиляция** — темат. серия клипов (стиль «с камер», топ-10 и т.п.),
  единый оверлей (CAM/таймкод/REC) склеивает разнородное в цельный эфир.
- Нормализация сегментов обязательна в любом формате (scale increase + crop под рамку).

## 0. Вводные (спросить один раз, коротко)

Объект (что снимаем), сценарий/настроение, длительность (15 ≈ 3 клипа / 30 с ≈ 6 клипов),
музыка (файл или «подбери»), текст поверх (слоганы / характеристики / свой), CTA
(призыв + промокод — код должен реально существовать в продукте, реклама не врёт).
Если в репозитории есть `BRANDBOOK.md` / `CLAUDE.md` с правилами стиля — читать их
и НЕ переспрашивать то, что там записано.

## 1. Стиль-паспорт (если брендбука нет — собрать из контекста)

Быстро извлечь из сайта/проекта: палитру, тон (дерзкий/премиум/фан), лексику, шрифтовую
эстетику, запретные штампы. Записать 5–8 строк «паспорта стиля» и держать его во всех
промптах и титрах. Банлист хук-артефактов всегда: «СТОП, не листай», «99% не знают»,
«досмотри до конца», кликбейт без правды.

## 2. Сценарий: хук от кульминации → биты → CTA

- **Хук = самый сочный кадр первой секундой** (не завязка!).
- Максимум 4–5 битов: один бит = один клип = одна мысль = одна строка титра.
- Финал — CTA-плашка (последние ~4.5 с): призыв + код/ссылка.
- Тренды, которые работают: before/after-трансформация через глитч, POV/дэшкам/бодикам
  эстетика («сборник с камер»), фейл-юмор в финале, макро-фетиш текстур, кадр «из-под
  капота» с пульсацией в такт басам, толпа с фонариками телефонов.

## 3. Генерация клипов (Higgsfield MCP, если подключён)

- Модель `kling3_0_turbo`, `aspect_ratio: "9:16"`, duration 5, ~7.5 кредитов/клип.
- В промпт всегда: `Vertical 9:16 reel shot`, стиль-паспорт, `cinematic film grain, no text`.
- При preset-notice — повторить с `declined_preset_id` из ответа.
- **Банк камер — в одном рилсе ракурсы НЕ повторяются**: низкий чейз у бампера · FPV-дрон ·
  вертолёт/кран · статика с проездом впритирку · кокпит/руки · отражение в зеркале/витрине ·
  макро · уровень асфальта · long-lens издалека · whip-pan · камера на преследователе ·
  переезд через объектив. Локации тоже варьировать.
- **Консистентность объекта — закон**: (1) канонический кадр → image-to-video через
  `medias role start_image`, (2) дословный «паспорт» объекта в каждом промпте
  (цвет, форма, детали), (3) клип с «чужим» объектом — перегенерить, в монтаж не пускать.
- Опрос: `job_status sync:true`; скачивать `Invoke-WebRequest` в scratchpad.

## 4. Монтаж (ffmpeg, filter_complex_script из файла)

- **Динамическая нарезка**: каждый 5-с клип резать на 2 куска ~2–2.4 с и перемешивать
  ракурсы (12 сегментов ≈ 26.4 с). Жёсткие склейки живее кроссфейдов; глитч-переходы
  (`xfade=pixelize|hlslice`) — только если склеек мало.
- **Нормализация КАЖДОГО сегмента** (иначе один «лежачий» клип кладёт весь ролик):
  `scale=720:1280:force_original_aspect_ratio=increase,crop=720:1280,fps=24,settb=AVTB`.
- **Грейд-пресеты**: underground — `eq=contrast=1.16:brightness=-0.03:saturation=0.78:gamma=0.96,
  colorbalance=rs=0.10:rm=0.04:bs=-0.04,vignette=PI/4.2,noise=alls=5:allf=t`;
  police-cam — saturation 0.62, `colorbalance=rs=0.04:bs=0.06`, noise 7 + оверлеи
  `CAM 07`, `timecode=...:rate=24`, мигающий `● REC` (`enable='lt(mod(t,1),0.6)'`).
- **Титры** (Impact, глиф `▸` ненадёжен — разделитель `//`):
  прыжки размера («ближе-дальше»): три drawtext с фазами `lt(mod(t-a,0.9),0.3)` и т.д.;
  глитч-вспышки: красная копия `x-7,y+2` + голубая `x+7,y-2`, `enable='lt(mod(t+φ,0.71),0.11)'`,
  рисуются ПОВЕРХ белого; полосы помех drawbox по слою и прямо по строке текста.
- **CTA-плашка**: код в красном box (`box=1:boxcolor=…@0.92`), пульс размером через
  две фазы enable.
- **Музыка**: `atrim`, `afade` in/out, `loudnorm=I=-14:TP=-1.5:LRA=11` (стандарт стриминга).
- Кодек: `libx264 -preset slow -crf 26 -pix_fmt yuv420p -c:a aac -b:a 128k -movflags +faststart`.
  Зерно дороже всего по битрейту: noise ≤5 держит 26 с в ~5 МБ.

## 5. Грабли (уже собраны — не повторять)

- PowerShell `Get-Content|-replace|Set-Content` УБИВАЕТ кириллицу в UTF-8 фильтрах —
  фильтр-скрипты только целиком через Write-инструмент.
- Кириллица в фильтре — файл строго UTF-8; шрифты: `C\:/Windows/Fonts/impact.ttf`
  (глитч-титры), `consola.ttf` (таймкоды/HUD).
- **ЗАПРЕТ: 16:9 внутри 9:16.** Генератор может отдать горизонтальную сцену с чёрными
  полосами внутри вертикального кадра — это БРАК, в монтаж не пускать, клип перегенерить.
  Профилактика в КАЖДОМ вертикальном промпте: `tall vertical composition, camera strictly
  upright, subject fills the frame vertically, no letterboxing, no black bars`.
  Контроль до монтажа: вытащить первый кадр каждого клипа (`ffmpeg -frames:v 1`) и
  посмотреть на полосы; нормализация (п.4) обязательна, но полосы она не лечит.
- `colorbalance` не имеет опции `ms` (только rs/gs/bs, rm/gm/bm, rh/gh/bh).
- `drawtext fontsize` не анимируется — «зум» делать фазами enable.
- Файл >30 МБ = перекрутил noise/CRF.

## 6. Выдача

Итог: `<имя>.mp4` 720×1280 (для соцсетей можно апскейл до 1080×1920 через
`upscale_video`). Если проект ведёт полку рилсов (например `src/data/reels.ts`) —
прописать туда и в CHANGELOG; иначе просто отдать путь к файлу и краткий разбор
(сцены, титры, сколько кредитов съедено).
