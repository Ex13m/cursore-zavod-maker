# /com — Справочник команд Claude Code (2026)

Когда пользователь вызывает `/com`, выведи справочник команд Claude Code.

## Как отвечать

- Без аргумента (`/com`) — покажи краткий оглавительный список всех 10 категорий + встроенные навыки, чтобы пользователь сориентировался.
- С аргументом-категорией или ключевым словом (`/com контекст`, `/com модели`, `/com git`) — выведи только релевантный раздел.
- Если пользователь описывает задачу («как откатить код», «как сжать историю») — найди подходящую команду и объясни её.

Отвечай по-русски, кратко, без лишней воды. Команды и флаги CLI оставляй в оригинале.

## 01. Управление контекстом и общение

| Команда | Назначение |
|---|---|
| `/clear` (`/reset`, `/new`) | Очищает переписку, начинает сессию с чистого листа. Используй при переключении на новую задачу — старый контекст ухудшает качество. |
| `/compact [инструкции]` | Сжимает историю в краткую сводку. Самая важная команда. Применяй проактивно при длинном контексте. Можно указать фокус: `/compact сохранить схему БД и обработку ошибок`. |
| `/context` | Визуализирует использование контекста цветовой сеткой и предлагает оптимизацию. |
| `/fork [имя]` | Ветвление переписки в текущий момент. Для исследования альтернатив без потери прогресса. |
| `/rewind` (`/checkpoint`) | Отматывает переписку и/или код к предыдущему состоянию. Спасает при неверном пути. |
| `/export [файл]` | Экспорт переписки в текстовый файл. Без имени — копирование/сохранение. |
| `/copy` | Копирует последний ответ в буфер. Для блоков кода — интерактивный выбор фрагментов. |

## 02. Модели и производительность

| Команда | Назначение |
|---|---|
| `/model [модель]` | Переключение модели в сеансе. Стрелки ←→ — уровень усилий. Паттерн: Sonnet для рутины, Opus для сложного. |
| `/fast [on\|off]` | Быстрый режим Opus — ускоренный вывод без смены модели. |
| `/effort [low\|medium\|high\|max\|auto]` | Глубина размышлений. low/medium/high сохраняются между сессиями. max — только Opus, в рамках сессии. auto — сброс. |

## Пользовательские навыки — самое мощное

Помести markdown-файл в `~/.claude/skills/имя-команды/SKILL.md` — и он автоматически станет slash-командой. Инструкции загружаются из файла, Claude выполняет описанный процесс.

Примеры от сообщества: `/commit` для commit-сообщений, `/pr` для описаний pull request, `/fix-pipeline` для анализа упавших CI и авто-исправления.

Формат Skills поддерживает frontmatter: разрешённые инструменты, модель выполнения, авто-запуск при релевантном контексте, создание субагента.
